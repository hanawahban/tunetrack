# AGENTS.md

Source of truth for any agent (or human) working in this repo. `CLAUDE.md` just points here — keep this file current instead of maintaining two.

## What this is

TuneTrack — a Last.fm-style scrobbling app styled as a used-record shop. See `PRODUCT.md` for the product vision and `DESIGN.md` for the visual design system. This file covers layout, running it, and code conventions.

## Layout

```
backend/            NestJS API
  src/
    <resource>/      one module per resource: albums, artists, tracks, users, auth, scrobbles, stats
      <resource>.controller.ts
      <resource>.service.ts
      <resource>.module.ts
      dto/            request DTOs — always real classes, never Partial<T> (see Rules)
    common/
      guards/         JwtAuthGuard, RolesGuard
      decorators/     @Roles(), @CurrentUser()
    prisma/           PrismaService (global module, injected everywhere)
    generated/prisma/ Prisma client output — gitignored, `npx prisma generate` to (re)create it
  prisma/
    schema.prisma     source of truth for the DB shape
    migrations/       one dir per migration, generated by prisma, never hand-edited
    seed.ts           creates the 3 test accounts (see below)
frontend/            React 19 + Vite
  src/
    pages/            route-level components
    components/
      ui/             shadcn/Radix primitives — use these, don't hand-roll buttons/dialogs/inputs
      records/        feature components (album/track dialogs, vinyl sleeve, etc.)
      layout/         shell/nav
    lib/
      api.ts          hand-written fetch wrapper + types — see Known in-flight work below
      auth-context.tsx  session state from decoded JWT (display only, not authorization)
docs/openapi.json     generated OpenAPI spec (backend writes it on boot) — committed, regenerated whenever the API changes. Not yet on master; see feature/swagger-api-docs.
```

## Running it

Full walkthrough with troubleshooting is in `README.md`. Condensed:

```bash
# backend
cd backend && npm install
cp ../.env.example .env   # set DATABASE_URL, JWT_SECRET
npx prisma generate && npx prisma migrate deploy && npx prisma db seed
npm run start:dev          # :3000, Swagger at /docs or /api/docs depending on branch

# frontend
cd frontend && npm install
cp .env.example .env
npm run dev                 # :5173
```

Test accounts (password `password123` for all): `admin@test.com`, `curator@test.com`, `listener@test.com`.

Type-check (there is no test suite yet — this is the current correctness gate):
```bash
cd backend && npx tsc --noEmit -p tsconfig.json
cd frontend && npx tsc -b
```

## Conventions

- **Package manager**: npm only. Both `backend/` and `frontend/` commit `package-lock.json` — don't introduce a pnpm/yarn lockfile.
- **DTOs are always real classes.** Never type a controller's `@Body()` as `Partial<CreateXDto>` — TypeScript mapped types erase to `Object` at runtime, and Nest's global `ValidationPipe` silently skips validation for non-class types. Write a dedicated `UpdateXDto` class instead (every resource already has one).
- **Auth pattern**: `@UseGuards(JwtAuthGuard)` at the controller class level; `@UseGuards(RolesGuard)` + `@Roles(Role.ADMIN, Role.CURATOR)` at the method level for mutating routes. Read routes (`GET`) are open to any authenticated role.
- **3-role RBAC**: `LISTENER` (browse + scrobble only), `CURATOR` (+ full CRUD on artists/albums/tracks), `ADMIN` (+ `PATCH /users/:id/role`). ADMIN and CURATOR are otherwise functionally identical — there's no other ADMIN-only route.
- **Authorization is server-side only.** The frontend decodes the JWT client-side (`lib/jwt.ts`) purely to display role/gate UI controls — never treat that as the actual authorization boundary; the guards are.
- **Services throw `NotFoundException`** for missing records rather than letting Prisma's raw error surface (P2025/FK violations) as an unhandled 500.
- **Env sync**: backend's `WEB_ORIGIN` and frontend's `VITE_API_URL` must point at each other's real address/port or CORS breaks. If Vite bumps off 5173 (port taken), either free it or update `WEB_ORIGIN` and restart the backend.
- **Commits**: Conventional Commits (`feat(scope): ...`, `fix(scope): ...`, `docs: ...`). No AI-attribution footers in commits, PR titles/bodies, or comments.
- **Branching**: one feature branch per issue, named for the issue/ticket where one exists.
